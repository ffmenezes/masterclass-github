# O que é Git e GitHub explicado com bananas | Coisas que você precisa saber!

## A tragédia do macaco programador

O que é **Git** e **GitHub**? O macaco tem um defeito grave e perigoso. Ele confia na própria memória e, pior, ele confia na sorte.

O macaco passa uma semana inteira escrevendo um código lindo. É uma plantação de bananas perfeita, com irrigação automática e espantalho anti-pássaros. Ele está orgulhoso. Aí, na sexta-feira à noite, cansado e com sono, ele decide mudar só uma coisinha no sistema de irrigação. Ele muda a cor da banana.

O código quebra, a água para de descer. Ele tenta desfazer, o código quebra mais, as árvores começam a pegar fogo. Ele entra em pânico, tenta voltar para como estava na terça-feira, mas ele não lembra o que escreveu na terça-feira.

Ele olha para a área de trabalho do computador e vê o cemitério da vergonha. Pastas chamadas `Projeto_final`, `Projeto_final_final`, `Agora_vai`, `Projeto_final_V3_oficial`, `Projeto_pelo_amor_de_Deus_funcionou`. Ele tenta abrir o backup, mas salvou o arquivo errado em cima do certo. O trabalho de uma semana virou lixo digital.

O macaco chora. O macaco quer voltar para a floresta e nunca mais tocar em um teclado.

## A magia antiga: Git e GitHub

Isso acontece porque o macaco vive no presente, mas o código, o código precisa viver na história. Para não perder as bananas e nem a sanidade mental, os humanos inventaram uma magia antiga e poderosa, uma verdadeira máquina do tempo. O nome dessa magia é **Git**. E o cofre dourado onde guardamos essa máquina e nossos tesouros se chama **GitHub**.

Se você não usa isso, sinto-lhe informar, você não é um programador. Você é apenas um homem das cavernas digital brincando com fogo dentro de um depósito de pólvora.

## Git e GitHub não são a mesma coisa

Primeiro, vamos tirar a confusão da cabeça do macaco iniciante. **Git** e **GitHub** são a mesma coisa? Achar que são iguais é como achar que a câmera fotográfica e o Instagram são a mesma coisa.

O **Git** é a câmera. É uma ferramenta, um programa que você instala no seu computador, na sua caverna escura. Ele serve para tirar fotos do seu código. Você escreve um pouco, o Git tira uma foto. Escreve mais um pouco, tira outra foto. Se você fizer uma besteira colossal, você pega a foto antiga e puf, o erro desaparece. Você volta no tempo antes do incêndio começar.

Já o **GitHub** é o Instagram, é a rede social, é o museu. É para lá que você manda as fotos do seu código que o Git tirou. O Git salva no seu PC, o GitHub salva na nuvem, no templo sagrado das bananas. Se o seu computador pegar fogo, ser roubado, cair no rio, a banana continua lá, segura, brilhando no GitHub. Você compra um computador novo, baixa tudo de volta e continua como se nada tivesse acontecido.

## Os três comandos sagrados

Mas como funciona essa mágica na prática? O macaco precisa entender o ritual. Imagine que você está preparando uma cesta de bananas para exportação. Você tem três comandos sagrados: o **add**, o **commit** e o **push**.

### git add

Primeiro, o **git add**. O macaco descascou 10 bananas. Ele olhou para elas. Cinco estão lindas, cinco estão podres. Ele vai salvar as podres? Não. O `git add` é o momento em que o macaco escolhe quais bananas vão para a cesta. É a área de preparação. Você diz: "Essa banana aqui entra. Essa casca velha fica de fora." O macaco coloca as bananas escolhidas na cesta. Mas a cesta ainda está aberta. Nada foi enviado ainda.

### git commit

Depois vem o comando mais importante da vida do programador, o **git commit**. O **commit** é o ato de fechar a cesta, passar a fita adesiva e colar um selo de garantia. Mas o Git é exigente. Ele não deixa você fechar a caixa sem um bilhete. Esse bilhete é a mensagem do **commit**.

O macaco precisa escrever o que tem dentro da caixa. Um macaco burro escreve "alterações" ou "arrumei coisas". Isso não serve. Daqui a três meses, você não vai saber que coisas foram essas. Um macaco sábio escreve: "Adicionei a função de descascar bananas automaticamente e removi as pontas pretas."

Pronto, você carimbou a caixa. Esse momento do tempo agora existe para sempre, gravado na pedra da história do seu projeto. Isso é um save point de videogame, só que muito mais poderoso.

### .gitignore: o guardião dos segredos

Mas espere, existe um perigo. O macaco tem um diário secreto onde guarda as senhas do banco e o mapa do tesouro. Ele quer mandar esse diário para o **GitHub**? Claro que não.

Para isso existe o guardião chamado **.gitignore**. O **.gitignore** é uma lista negra, um pedaço de papel onde você escreve tudo que o Git deve fingir que não vê. Você escreve lá senhas, fotos constrangedoras, arquivos de configuração secretos. O Git passa por esses arquivos e, como um segurança leal, vira a cara e ignora. Isso salva o macaco de ser hackeado.

### git push

Agora que a caixa está fechada e segura, o macaco precisa mostrar isso pro mundo. Ele usa o terceiro comando, **git push**. **Push** significa empurrar. Você empurra sua cesta de bananas da sua caverna direto para a nuvem. Quando o código chega no **GitHub**, a mágica se completa, está salvo.

## Universos paralelos: branches e merge

Agora vamos falar sobre universos paralelos. Essa é a parte que separa os meninos dos gorilas prateados.

O macaco quer tentar uma ideia maluca. Ele quer tentar criar uma banana quadrada com gosto de morango. Se ele mexer na árvore principal, aquela que dá o sustento da tribo, que chamamos de **branch main** ou **master**, ele corre um risco enorme. Se a experiência der errado, ele mata a árvore principal. O site cai, o cliente grita, o chefe demite o macaco.

O macaco inteligente não mexe na árvore sagrada. O macaco inteligente cria uma **branch**, um ramo, uma linha do tempo alternativa, o multiverso da loucura. Nessa realidade paralela, o macaco faz o que quiser. Ele pinta a banana de azul, coloca gosto de pimenta, vira o código do avesso. A árvore principal nem sente. Ela continua lá, produzindo bananas amarelas normais.

Se a experiência da banana quadrada der errado e explodir, o macaco simplesmente corta aquele galho. Puf! Acabou. Ninguém se feriu.

Mas e se der certo? E se a banana quadrada for deliciosa? Aí o macaco faz o ritual da união, o **merge**. Ele pega aquela linha do tempo alternativa e funde com a linha do tempo principal. Agora a árvore oficial produz bananas quadradas. A evolução aconteceu sem riscos.

## Trabalho em bando: conflitos e Pull Requests

Mas o **GitHub** não serve só para guardar código, ele serve para trabalhar em bando. Imagine uma tribo inteira mexendo na mesma árvore. Um macaco muda a casca, o outro muda a polpa ao mesmo tempo.

O **Git** é o juiz dessa bagunça. Se dois macacos tentam mudar a mesma banana ao mesmo tempo, o Git grita "conflito". A tela fica vermelha, símbolos estranhos aparecem. Isso parece assustador, mas é a salvação. O Git impede que o macaco A apague o trabalho do macaco B sem querer. Ele para tudo, cruza os braços e diz: "Ei, vocês dois, decidam qual versão vale: a banana descascada ou a banana frita?" Vocês conversam, resolvem o conflito e a paz reina na selva.

E como você sabe se o código do outro macaco é bom? Você não deixa ele fazer o **merge** direto. Isso é perigoso. Você exige um **Pull Request**.

O **Pull Request** é um pedido formal. O macaco diz: "Ó grande conselho dos anciãos, eu criei esta banana quadrada. Posso misturar na árvore sagrada?" Os outros macacos olham o código. Eles julgam, eles criticam: "Essa banana está torta. Faltou açúcar aqui." Isso é o **code review**. Só depois que os anciãos aprovam, o código entra. Isso garante que nenhuma banana podre envenene a tribo.

## Open Source e Fork: o shopping center do código

Além disso, o **GitHub** é o maior shopping center de código do universo. Existe uma coisa chamada **open source**, código aberto.

Imagine que um gorila sábio do outro lado do mundo criou uma máquina incrível de descascar cocos. Ele colocou o código no **GitHub** e deixou aberto. Você pode ir lá e fazer um **fork**. O **fork** é basicamente você pegar a máquina dele, fazer uma cópia exata para sua caverna e usar. E se você melhorar a máquina dele, você pode mandar a melhoria de volta.

É assim que a tecnologia humana evolui. Um monte de macacos colaborando, copiando e melhorando as ferramentas uns dos outros.

## O segredo da contratação: o calendário verde

Por fim, o segredo da contratação. Nenhum recrutador contrata macaco pelo que ele diz que sabe. Papagaio também fala. Contratam pelo que ele mostra que sabe.

O seu perfil do **GitHub** tem um calendário, um gráfico cheio de quadradinhos. Cada dia que você faz um **commit**, cada dia que você manda código, um quadradinho fica verde. Um perfil cheio de quadradinhos verdes brilhantes é um sinal visual poderoso. Ele grita: "Este macaco trabalha! Este macaco coda todo dia. Este macaco é disciplinado."

Já um perfil cinza, vazio e triste é um sinal de alerta. Macaco preguiçoso, macaco que só estuda, mas não pratica.

O **GitHub** é o seu portfólio, é a sua vitrine. Não adianta colocar no currículo "conhecimento avançado em Java". Mostre o **repositório**. Mostre o código. Mostre a estrutura das suas pastas. Mostre que você sabe usar **branches**, que sabe escrever mensagens de **commit** decentes.

## Conclusão: instale o Git, pinte seu calendário de verde

**Git** e **GitHub** parecem monstros de sete cabeças no começo. É muita tela preta, muito comando estranho: `git`, `git status`, `git checkout`. Parece que você está hackeando a NASA, mas não se engane, é apenas uma ferramenta de organização. É só uma máquina do tempo e um álbum de figurinhas. É a ferramenta que garante que o suor do seu trabalho não será desperdiçado por um erro bobo ou um computador quebrado.

Se você quer ser um programador primata de elite, pare agora mesmo de salvar arquivos com nomes aleatórios no seu desktop. Instale o **Git**. Crie uma conta no **GitHub**. Faça seu primeiro **repositório** hoje. Pinte seu calendário de verde.

O mercado não quer saber se você decorou a sintaxe de todas as linguagens. O mercado quer saber se você sabe trabalhar em equipe, se sabe guardar seu progresso e se sabe compartilhar suas bananas com a tribo.

Não deixe seu código morrer esquecido no escuro do seu HD. Mande ele para as estrelas. Faça o **push** para o sucesso.

Se inscreva no canal para evoluir de macaco codificador para gorila arquiteto. Salve seu progresso. Dê o **commit** na sua...
