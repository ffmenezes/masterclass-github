# GIT x GitHub — Parte 1

## Introdução

Rapaziada, isso aqui parece que é um bicho de sete cabeças, mas depois que vocês cobrirem, vocês vão ver que é mais fácil da mão. **Git**, **GitHub** e assim. A gente vai trabalhar aqui, eu pensei, e a gente trabalha como se a gente fosse criando um sistema de uma calculadora. Uma coisa bem simples, bem básica, tá ligado?

## O problema do versionamento manual

Isso da computação, ficar fazendo vários arquivos de versionamento, tipo, versão um, versão dois, fica confuso. Onde você fez modificações? Onde você deixou de fazer modificações? Porque você vai se perder do seu código.

**Git** facilita muito, muito, muito o trabalho em equipe. Vocês vão trabalhar — só que faz sozinho — um monte de gente junto, mexendo junto, fazendo junto. A gente consegue trabalhar no mesmo sistema, as pessoas conseguem trabalhar no mesmo sistema juntos, de batalha organizada em casa.

## Conflitos em equipe

Como é que eu estou dito aqui? É a questão de tipo, você, como é que ficou as questões antigas e os conflitos? Às vezes, como é que a gente tá trabalhando em equipe. Vou dar exemplo aqui, o Yuri tá aqui. Conflitou tudo, tá ligado? Tipo, depois, qual que é a versão certa? Foi aqui o **código** ou o que o Yuri **codou**? Fudeu.

Pra resolver esses conflitos, o **Git** também ajuda muito, muito com isso.

## O que é controle de versão

O que é controle de versão? A gente pode imaginar ele como se fosse uma máquina do tempo. A gente guarda nessa máquina do tempo. Ele é um sistema de controle de versão. Ele é um sistema de controle de versão **distribuído**.

Os fatores mais antigos antes do **Git** faziam com que você tivesse um **repositório central**. As pessoas ficavam dentro do projeto que estava naquele **repositório** central. O **GitHub** isso, e você consegue ter o projeto inteiro na sua máquina. Isso é a grande vantagem do **Git**: isso é localmente.

Outra coisa: o sistema de controle de versão, você consegue muito fácil rastrear as mudanças e fazer com que você tenha a colaboração de outras pessoas trabalhando nisso.

## A história do Git e do Linux

Existem várias razões do porquê ele criou o **Git**, mas tem várias histórias que contam o porquê ele criou o **Git**. Mas, acredita, motivo: ele estava desenvolvendo o **Linux** com outras pessoas.

O que é do **Linux** com outras pessoas? O desenvolvedor aberto, ele era gratuito. Ajudando a desenvolver isso. Como é que esse sistema funcionava? Como é que a empresa se ligou que esse cara tava tentando, tipo, estudar o sistema deles, eles foram lá e bloquearam — tipo assim, agora não tá mais gratuito.

Tava ajudando a desenvolver o **Linux**, ficou tipo: e agora, gente, bagulho da frente. Eu vou criar o meu próprio versionador. E em quase duas semanas o cara foi lá e criou o **Git**, né? Como a gente tem hoje. Ele focou em ser **rápido**, **seguro** e **distribuído**. Ter várias pessoas conseguindo trabalhar e várias pessoas conseguindo ter o projeto inteiro na mão.
